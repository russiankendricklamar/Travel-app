### Partial Requirements Detail

**OFFL-02**: Cache-first lookup works for route *display* — cached polylines render correctly offline. However, auto-cached routes (stored during online browsing) do not carry `navigationSteps`. When offline, `MapViewModel.startNavigation()` attempts `MKDirections` fallback which fails, and `guard !navigationSteps.isEmpty` silently aborts navigation. Routes pre-cached via the "ПОДГОТОВИТЬ ОФЛАЙН" button are unaffected (they correctly store steps via `preCacheDay`).

**OFFL-04**: The `offlineNoCacheMessage` ("Маршрут недоступен офлайн") view is unreachable dead code — the condition `vm.activeRoute == nil` is nested inside `if let route = vm.activeRoute { }`, making it permanently false. The explanatory message for offline+no-cache state never renders. Other degradation behaviors (carousel hidden, pills disabled, reroute warning) work correctly.
