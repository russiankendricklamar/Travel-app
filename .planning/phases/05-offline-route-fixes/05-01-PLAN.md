<task id="2" name="Store navigationSteps in auto-cached routes">
<read_first>
- Travel app/Services/RoutingService.swift (lines 195-231, the Place-UUID calculateRoute overload)
- Travel app/Services/RoutingService.swift (fetchNavigationSteps method)
- Travel app/Services/OfflineCacheManager.swift (preCacheDay method — reference pattern for step fetching)
</read_first>

<action>
In RoutingService.swift, the `calculateRoute(fromPlace:toPlace:mode:tripID:context:)` method (line 196) stores `results.first` to cache but that RouteResult has empty `navigationSteps` because `parseRoutesAPIResponse` never populates them.

Fix: After getting results from the online API, fetch navigation steps before storing to cache. Add step fetching between the API call and the store call:

```swift
if await OfflineCacheManager.shared.isOnline {
    let results = await calculateRoute(
        from: origin.coordinate,
        to: destination.coordinate,
        mode: mode
    )
    // Fetch and attach navigation steps before caching
    if var first = results.first {
        let steps = await fetchNavigationSteps(
            from: origin.coordinate,
            to: destination.coordinate,
            mode: mode,
            existingTransitSteps: first.transitSteps
        )
        first = RouteResult(
            polyline: first.polyline,
            distance: first.distance,
            expectedTravelTime: first.expectedTravelTime,
            transportType: first.transportType,
            transitSteps: first.transitSteps,
            navigationSteps: steps,
            staticDuration: first.staticDuration
        )
        await RoutingCacheService.shared.store(
            first, origin: origin.id, dest: destination.id,
            tripID: tripID, context: context
        )
        return [first] + results.dropFirst()
    }
    return results
}
```

This mirrors the pattern used in `OfflineCacheManager.preCacheDay` which correctly fetches steps before storing.

Note: Check RouteResult init signature — if it uses different parameter names or has a different initializer, adapt accordingly. RouteResult may need to be examined to confirm exact init parameters.
</action>

- The method still returns `results` to the caller (no change in return behavior)
- Build succeeds
