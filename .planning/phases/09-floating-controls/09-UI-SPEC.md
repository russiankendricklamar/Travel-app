## Spacing Scale

Declared values — multiples of 4, aligned to `AppTheme` constants:

| Token | Value | SwiftUI Constant | Usage |
|-------|-------|-----------------|-------|
| xs | 4pt | `AppTheme.spacingXS` | Icon-internal gaps |
| sm | 8pt | `AppTheme.spacingS` | Compass-to-container gap (D-02) |
| md | 16pt | `AppTheme.spacingM` | Trailing edge padding from screen edge (D-36) |
| lg | 24pt | `AppTheme.spacingL` | — |
| xl | 32pt | `AppTheme.spacingXL` | — |

**Source:** CONTEXT.md D-02, D-36 + AppTheme.swift

Exceptions:
- Container bottom offset: **88pt** (chosen at midpoint of D-37 range 80-90pt, aligns with current `.padding(.bottom, 90)` at TripMapView:179 minus 2pt for visual tightness)
- Button hit area: **44pt × 44pt** (Apple HIG minimum touch target, D-06)
- Container width: **44pt** (D-05 — buttons fill container with no internal padding)
- Divider thickness: **0.5pt** (D-04 — matches project `AppTheme.borderWidth`)

## Component Inventory

### FloatingControlsOverlay (new view)

Top-level overlay positioned in TripMapView ZStack, replacing the existing floating location button (TripMapView:149-181).

**Structure:**
```
VStack(spacing: 0) {
    // 1. Native MapCompass — auto-hides when north-facing
    MapCompass(scope: mapScope)          // D-09, D-11

    Spacer().frame(height: 8)            // D-02: 8pt gap

    // 2. Blur container with 3 buttons
    VStack(spacing: 0) {
        TransitButton                     // D-13..17
        Divider (0.5pt, white 15%)       // D-04
        ElevationButton                   // D-18..22
        Divider (0.5pt, white 15%)       // D-04
        LocationButton                    // D-23..26
    }
    .frame(width: 44)
    .background(.ultraThinMaterial)
    .clipShape(RoundedRectangle(cornerRadius: 12, style: .continuous))   // D-03
    .shadow(color: .black.opacity(0.35), radius: 14, y: 4)              // D-07
    .preferredColorScheme(.dark)
}
.padding(.trailing, 16)                  // D-36
.padding(.bottom, 88)                    // D-37 — resolved to 88pt
.frame(maxWidth: .infinity, maxHeight: .infinity, alignment: .bottomTrailing)
.opacity(isVisible ? 1 : 0)             // D-27..31
.animation(.spring(response: 0.35, dampingFraction: 0.85), value: isVisible)  // D-28
```

**Visibility condition (`isVisible`):**
```swift
vm.sheetDetent == .peek && !vm.isNavigating && !vm.showPrecipitation && !isOfflineWithCache
```
Source: CONTEXT.md D-27, D-29, D-30, D-31. Uses snap-by-detent (not drag offset) as specified in D-27.

## State Management

Two new `@Observable` properties needed on `MapViewModel`:

| Property | Type | Initial | Controls |
|----------|------|---------|---------|
| `showTraffic` | `Bool` | `true` | `.mapStyle(showsTraffic:)` |
| `show3DElevation` | `Bool` | `false` | `.mapStyle(elevation:)` |

**mapStyle binding:**
```swift
.mapStyle(.standard(
    elevation: vm.show3DElevation ? .realistic : .flat,
    pointsOfInterest: .including([.museum, .nationalPark, .park, .restaurant]),
    showsTraffic: vm.showTraffic
))
```

Existing `showsTraffic: true` hardcoded at TripMapView:414 is replaced by `vm.showTraffic`.
