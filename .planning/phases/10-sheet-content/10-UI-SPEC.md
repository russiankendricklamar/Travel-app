## Typography

Four sizes in use — matched to existing codebase values (do not introduce new sizes).

Exactly 2 font weights: **Medium (500)** and **Semibold (600)**.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Section header | 13pt | Semibold (600) | 1.2 | "Сегодня · Город", "Карта" labels |
| Body / place name | 15pt | Medium (500) | 1.4 | Place names, completer titles |
| Search / chip label | 14pt | Medium (500) | 1.3 | Chip text labels |
| Map control label | 10pt | Medium (500) | 1.2 | Labels under circle buttons (Слои, Осадки, etc.) |

Source: measured from `MapSearchContent.swift` lines 137, 189, 465, 430, 596.

Note on map control labels: the pre-existing codebase used Regular (400) at 10pt. Medium (500) at 10pt is visually indistinguishable at small sizes and maintains the 2-weight constraint without visual harm. Phase 10 may update these labels to Medium during implementation; if existing file uses `.regular`, it is acceptable to leave as-is since the visual difference is imperceptible — but no new Regular (400) usage should be introduced.

Maximum 2 weights per component and across the full spec:
- Body and labels: Medium (500)
- Headers: Semibold (600)

All colors derived from the existing `AppTheme` system. The sheet lives in dark color scheme (established in Phase 7).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `.systemBackground` adaptive (dark: ~#1C1C1E) | Sheet background — set by Phase 7 geometry |
| Secondary (30%) | `.quaternary.opacity(0.5)` | Chip unselected fill, map control circle fill, search bar inner background |
| Accent (10%) | `AppTheme.sakuraPink` (palette-aware) | Selected chip fill (see note), cancel button text, visited checkmark tint, cursor color |
| Destructive | not applicable — no destructive actions in Phase 10 | — |

Accent reserved for:
1. Selected category chip background (`Color(.label)` — inverted for contrast, NOT sakuraPink — see below)
2. Cancel button text color (`.foregroundStyle(AppTheme.sakuraPink)` — already implemented)
3. AI sparkles button when AI mode is active

Note on selected chip: existing code uses `Color(.label)` (adaptive black/white) for selected chip fill and `Color(.systemBackground)` for text — this high-contrast approach matches Apple Maps exactly. Do NOT change to sakuraPink.

Note on Осадки (precipitation) tint: `AppTheme.oceanBlue` is the active tint for the precipitation toggle. This is a semantic weather color, not the accent color. It is declared in the Interaction States table below. It is NOT reserved as an accent use case.

## Component Inventory

All components below are ALREADY IMPLEMENTED in `MapSearchContent.swift`. Phase 10 modifies visibility conditions and section ordering only — no new component construction.

### Category Chips Row

- Horizontal `ScrollView` with `HStack(spacing: 8)`, `.padding(.horizontal, 16)`
- Each chip: `Capsule()` background, 12pt H / 8pt V padding, SF Symbol icon 13pt + label 14pt medium
- Selected state: `Color(.label)` fill + `Color(.systemBackground)` text (inverted, high contrast)
- Unselected state: `Color(.label).opacity(0.12)` fill + `Color(.label).opacity(0.7)` text
- Loading state: `ProgressView().scaleEffect(0.6)` replaces icon while category search runs
- Chips defined in `MapViewModel.quickCategories`: Музеи (building.columns), Парки (tree), Магазины (bag), Отели (bed.double) + others as defined in ViewModel

### "Сегодня · [Город]" Section

- Section header: `HStack` with "Сегодня" + "·" + `today.cityName` all 13pt semibold `.secondary`, plus `visitedCount/total` counter right-aligned
- `Divider()` below header with 14pt horizontal padding
- Place rows: 28pt circle icon (`.quaternary` fill) + place name 15pt medium + visited checkmark right
- Visited: `checkmark.circle.fill` in `AppTheme.bambooGreen`; unvisited: `circle` in `.tertiary`
- Row tap: collapse sheet to `.peek` + set `vm.selectedPlaceID`
- `Divider()` between rows with 52pt leading inset (aligns to text after icon)
- Empty / no active trip: section not rendered (guarded by `vm.trip.todayDay` optional)

### Map Controls Row

- Section header: "Карта" 13pt semibold `.secondary`
- Horizontal `ScrollView` with `HStack(spacing: 12)`, `.padding(.horizontal, 16)`
- Each control: 36pt circle, `.quaternary.opacity(0.5)` fill, SF Symbol 15pt, label 10pt below
- Controls (left to right): Слои (Menu), Осадки (toggle), Обзор (discover nearby), Все места (zoom to all)
- Active tint (Осадки only): `AppTheme.oceanBlue` when `vm.showPrecipitation == true`

## Section Order and Visibility

### Half mode (search NOT focused, query empty)

Order top → bottom within the non-scrollable flat stack:

1. Category chips row (always visible)
2. "Сегодня · Город" section (only when `vm.trip.todayDay != nil && !today.sortedPlaces.isEmpty`)
3. Map controls row (always visible)

Rationale for D-04 (Claude's discretion): In Apple Maps, chips appear first as the primary discovery entry point. Today's places follow as the highest-value contextual content. Map controls are secondary utility — they appear last. Half mode has ~260pt of vertical space; chips (~44pt) + today section (variable, typically 2-4 places at ~50pt each) + map controls (~88pt) fits without scroll in the common case of 1-3 places today.

### Half mode (search focused OR query non-empty)

Completer suggestions replace chips/today/controls with a fade transition:
- `withAnimation(.easeInOut(duration: 0.2))` on the conditional showing chips/today/controls
- When `vm.searchQuery` is cleared: chips/today/controls fade back in with same 0.2s easing

### Full mode

Full mode wraps content in `ScrollView`. Section order:

1. Category chips row
2. "Сегодня · Город" section
3. Map controls row
4. "Недавние" section — full-mode-only placeholder (see Empty States below)
5. Search results / AI results / completer — replace all above when active

### Peek mode

No content sections visible. Only search bar handle area is shown (established in Phase 8).

## Scroll / Drag Strategy (D-05 — Claude's Discretion)

**Decision: No scroll in half mode. Scroll only in full mode (already implemented).**

Rationale: Half mode has ~260pt available below search bar. The content (chips + today section for typical 1-3 places + map controls) fits without scrolling. Introducing a ScrollView in half mode creates gesture conflict where the user's downward drag to dismiss could be captured by the ScrollView. The existing code already implements this correctly at lines 52-71 — the fix is only in the visibility condition at line 90.

In full mode: existing `ScrollView` implementation with `ScrollOffsetKey` preference key for scroll-aware divider is retained as-is.

If today's section exceeds 4 places in half mode: the section truncates display to 3 rows with a "Показать все (\(count))" button at 13pt semibold in `AppTheme.sakuraPink` that expands sheet to `.full`.

## Visibility Fix (D-01 — Locked Decision)

The ONLY change to content routing: remove the `isSearchFocused` condition on line 90.

Current (broken):
```swift
if isSearchFocused && vm.completerResults.isEmpty && vm.searchQuery.isEmpty,
   vm.sheetContent == .idle || vm.sheetContent == .searchResults {
```

Required:
```swift
if vm.completerResults.isEmpty && vm.searchQuery.isEmpty,
   vm.sheetContent == .idle || vm.sheetContent == .searchResults {
```

This makes chips + today + map controls visible in half mode immediately, matching Apple Maps behavior (D-01).

## Empty States (D-06 — Claude's Discretion)

| Scenario | Handling |
|----------|----------|
| No active trip | "Сегодня" section not rendered (guarded by `vm.trip.todayDay` optional). No placeholder needed — chips + map controls fill the space. |
| Active trip, no places today | "Сегодня" section not rendered (guarded by `!today.sortedPlaces.isEmpty`). No placeholder. |
| Offline (no map data) | Existing `OfflineBanner` capsule handles this at the app level — no additional handling needed in sheet content. |
| Full mode "Недавние" section (no history) | Section not rendered — no empty state placeholder. |

## Animation Contract

| Transition | Spec |
|-----------|------|
| Chips/today/controls appear (search cleared) | `.easeInOut(duration: 0.2)` opacity — use `.transition(.opacity)` already in place |
| Chips/today/controls disappear (search focus) | `.easeInOut(duration: 0.2)` opacity — matches appear timing |
| Scroll divider appear (full mode, offset > 2pt) | `.easeInOut(duration: 0.15)` — already implemented |
| Cancel button slide in | `.spring(response: 0.3)` move + opacity — already implemented |
| Sheet snap to peek on place tap | `.spring(response: 0.35, dampingFraction: 0.85)` — established in Phase 7 |
| "Показать все" expand to full | `.spring(response: 0.35, dampingFraction: 0.85)` — matches sheet transitions |

Note: Do NOT introduce new spring parameters. The response: 0.35 / dampingFraction: 0.85 pair is locked per STATE.md.
