## Color

Entire palette comes from `AppTheme.swift`. Phase 4 introduces one new semantic use: `bambooGreen` for the "done" state of the precache button.

| Role | AppTheme Token | Hex | Usage |
|------|---------------|-----|-------|
| Dominant (60%) | `.ultraThinMaterial` / `GradientBackground` | palette-adaptive | Bottom sheet backgrounds, floating card surfaces |

Accent (`sakuraPink`) reserved for:
- Precache button in idle state (background fill)
- Progress ring stroke and percentage text

`bambooGreen` reserved for:
- Precache button in done state only
- Day-cached status badge on TripMapView

`toriiRed` reserved for:
- "Очистить кэш маршрутов" button in Settings (destructive)
- Inline error foreground in no-cache offline message

## Component Inventory

New components introduced in this phase. All must use existing glass modifiers from AppTheme.swift.

### 1. OfflinePrecacheButton (NEW file: `Views/Shared/OfflinePrecacheButton.swift`)

Three visual states — single component with internal state machine:

| State | Visual | Interaction |
|-------|--------|-------------|
| `idle` | Capsule button, `sakuraPink` fill. Icon: `arrow.down.circle.fill`. Text: "ПОДГОТОВИТЬ ОФЛАЙН" (15pt semibold, .white, tracking 1). | Tap → triggers `preCacheDay`; disabled when `isOnline == false` (see disabled state below) |
| `idle (offline)` | Same capsule, `sakuraPink.opacity(0.4)` fill. Subtitle beneath: "Недоступно без сети" (11pt medium, `textSecondary`). | Tap does nothing (`.disabled(true)`). No CTA affordance. |
| `loading` | 44×44pt circular progress ring (6pt stroke, `sakuraPink` track on `sakuraPink.opacity(0.15)` background). Text to the right: "Загрузка маршрутов..." (11pt bold, `textSecondary`). Percentage below ring: "N%" (10pt bold, `sakuraPink`). | Tap disabled. Cancel not offered. |
| `done` | Capsule button, `bambooGreen` fill. Icon: `checkmark.circle.fill`. Text: "МАРШРУТЫ ГОТОВЫ" (15pt semibold, .white, tracking 1). | Tap re-triggers pre-cache (refresh). Badge on TripMapView day indicator also turns green. |

Ring animation: `.animation(.easeInOut(duration: 0.3), value: progress)` — exact pattern from PackingListView.
Button shape: `Capsule()` with `.padding(.horizontal, 20).padding(.vertical, 12)`.
Button placement: inside `TripMapView` bottom sheet idle state — above the route section header, floating-aligned to trailing edge with `.padding(.trailing, 16)`. Exact position is `Claude's Discretion` per CONTEXT.md.

Source: CONTEXT.md decisions, RESEARCH.md Pattern 6, PackingListView.swift (lines 130-136)

## Interaction States

| Element | States |
|---------|--------|
| OfflinePrecacheButton | idle / idle-offline-disabled / loading / done |
| Transport pills (offline) | enabled-cached / disabled-no-cache / loading (spinner during route calc) |
| Route alternatives carousel | visible (online) / hidden (offline) — no empty state, silently absent |
| Start navigation button | enabled (route loaded) / disabled (loading) — unchanged from Phase 2/3 |
| Settings cache clear | default / confirmation dialog / clearing (brief async) |

Transitions:
- idle → loading: immediate on tap, no animation delay
- loading → done: `.spring(response: 0.4, dampingFraction: 0.7)` scale transition on checkmark icon appearance
- loading → idle (on error): resets to idle state; error surfaced via `routeError` string in existing MapViewModel pattern

## Animation Contract

| Animation | Spec |
|-----------|------|
| Progress ring fill | `.easeInOut(duration: 0.3)` on `.trim(from: 0, to: progress)` value change |
| Button state idle→loading | `.easeInOut(duration: 0.2)` opacity crossfade |
| Button state loading→done | `.spring(response: 0.4, dampingFraction: 0.7)` on checkmark icon scale from 0.5→1.0 |
| Carousel hide (offline) | No animation — conditionally rendered, not animated out |
| Toast appear/dismiss | `.transition(.move(edge: .top).combined(with: .opacity))` with `.easeOut(duration: 0.25)` |

Source: RESEARCH.md Pattern 6, CONTEXT.md decisions, existing project patterns (withAnimation(.easeInOut(0.35)) from Phase 3)
