# Quick Task 260321-4u8: Map Search Pill Refactor Summary

**One-liner:** Capsule search field with focus-conditional dark category chips and clean offline indicator replacing button.

## Objective

Refactor map search UI for cleaner Apple Maps-like UX: pill-shaped search field, dark chip colors shown only on focus, offline precache button removed (indicator only), black bar on full sheet expansion fixed.

## Changes Made

### MapSearchContent.swift

- **Search field shape:** `RoundedRectangle(cornerRadius: 12)` replaced with `Capsule()` — true pill shape
- **Chips visibility:** Added `isSearchFocused &&` condition — chips now only appear when text field is focused
- **Chips animation:** Added `.transition(.opacity)` on chips and `.animation(.spring(response: 0.3), value: isSearchFocused)` on outer VStack
- **Selected chip color:** `AppTheme.sakuraPink` → `Color(.label)` (black in light mode, white in dark mode)
- **Unselected chip color:** `Color.primary.opacity(0.08)` → `Color(.label).opacity(0.12)`
- **Chip text color:** Unselected `.primary` → `Color(.label).opacity(0.7)`, selected stays `Color(.systemBackground)`

### TripMapView.swift

- **OfflinePrecacheButton removed:** Entire `else` branch removed — only `if` branch showing green checkmark indicator when day is cached
- **toolbarColorScheme:** Added `.toolbarColorScheme(.dark, for: .navigationBar)` to prevent transient dark chrome during sheet animation

### MapBottomSheet.swift

- **Safe area padding:** Added `let safeAreaTop = geo.safeAreaInsets.top` and `.padding(.top, detent == .full ? safeAreaTop : 0)` on the inner VStack — in full mode the sheet content starts below the status bar, eliminating the black bar overlap
