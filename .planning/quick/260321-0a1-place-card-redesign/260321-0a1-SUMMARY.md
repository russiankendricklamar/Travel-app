# Quick Task 260321-0a1: Place Card Redesign Summary

**One-liner:** Apple Maps-style place card with Look Around hero, left-aligned category badge header, circular action buttons, inline hours, and unified layout across all three detail modes.

## What Was Built

Complete rewrite of `MapPlaceDetailContent.swift` (~560 lines) implementing Apple Maps information hierarchy for the map bottom sheet.

### Hero Section

- `heroSection` ViewBuilder: Look Around as 200pt full-width primary visual (edge-to-edge, no corner radius or horizontal padding); first Google photo as fallback hero; loading placeholder with ProgressView
- `photoThumbnails`: 56x56pt rounded(8) horizontal scroll strip below hero. Shows all photos when Look Around is hero; skips first (already used as hero) when no Look Around scene

### Header

- `placeHeaderNew`: left-aligned HStack with 32pt circle category badge (sakuraPink 0.12 bg + category SF Symbol), VStack for name (21pt bold) + subtitle (14pt secondary), Spacer, and xmark close button — replaces centered ZStack layout

### Quick Status Row

- Inline capsules in horizontal ScrollView: open/closed (green/red bg 0.12), star rating with count, priceLevel mapped to ₽ signs (Free/₽/₽₽/₽₽₽/₽₽₽₽), visited checkmark badge

### Circular Action Buttons

- `circularButton`: VStack with 44pt Circle (filled=primary, outlined=tinted) + 11pt label below
- `placeActionButtons`: Маршрут (filled), Вызов, Сайт, AR (#if !simulator)
- `searchActionButtons`: transport Menu, Маршрут (filled), Вызов, Сайт

### Inline Hours

- `inlineHoursSection`: clock icon + "Часы работы" label + today's line always visible; chevron tap expands full 7-day schedule via `vm.showAllHours`. Today index = `(weekday + 5) % 7` to map Calendar.weekday (1=Sun) to Google (0=Mon)

### Merged Contact Info

- `mergedContactInfo` and `hasContactInfo`: single VStack — address prefers Apple Maps, phone/website each shown once (Apple first, Google fallback), Google Maps deep link row

### Reviews

- `reviewsSection`: first review always visible; "Все отзывы (N)" button expands remaining via `vm.showAllReviews`

### Unified Layout

- `unifiedDetailContent(name:subtitle:categoryIconName:actionButtons:extraContent:visitedBadge:)`: single shared function renders hero → thumbnails → header → status → actions → hours → contacts → reviews → extraContent
- `placeDetailView`, `searchItemDetailView`, `aiResultDetailView` each call unified with their unique actions/extra via AnyView closures

None - plan executed exactly as written.
