# Quick Task 260321-00h: Apple Maps-Style Search UX in Maps Summary

**One-liner:** MKLocalSearchCompleter instant typeahead replacing 400ms debounce, full-sheet pill tap, and "Отмена" cancel button matching Apple Maps search UX conventions.

## Objective

Replace the sluggish 400ms debounce + MKLocalSearch-per-keystroke mechanic with instant MKLocalSearchCompleter typeahead, and fix the sheet/keyboard UX to match Apple Maps: pill tap opens full sheet with keyboard focus, "Отмена" button replaces X for clean dismissal.

## Changes Made

### Task 1 — MapViewModel.swift

- **SearchCompleterDelegate** (private final class, NSObject + MKLocalSearchCompleterDelegate): holds `weak var viewModel: MapViewModel?`, forwards `completerDidUpdateResults` and `didFailWithError` callbacks to vm on MainActor via Task.
- **completerResults: [MKLocalSearchCompletion]** — observable array for live UI binding.
- **isCompleterActive: Bool** — computed: `!searchQuery.isEmpty && searchedItem == nil && sheetContent != .searchResults`.
- **init** wires up delegate: creates `SearchCompleterDelegate`, stores it in `completerDelegate`, assigns `searchCompleter.delegate`, sets `resultTypes = [.pointOfInterest, .address]`.
- **updateCompleterQuery()** — sets `searchCompleter.region` if visibleRegion available, sets `queryFragment`. Guards query length < 2 (clears results). No debounce — completer handles throttling internally.
- **selectCompleterResult()** — async, creates `MKLocalSearch.Request(completion:)`, resolves to first MKMapItem, calls `selectSearchResult(item)`. On error: fills searchQuery for manual submit fallback.
- **dismissSearch()** and **dismissDetail()** both clear `completerResults` and reset `searchCompleter.queryFragment = ""`.

### Task 2 — MapSearchContent.swift + TripMapView.swift

**MapSearchContent.swift:**
- `searchBar` property renamed to `searchFieldContent` (inner pill content only).
- `body` restructured: outer `HStack` wraps `searchFieldContent` + animated "Отмена" `Button` (foreground: `AppTheme.sakuraPink`, font 16pt, spring animation on appear/disappear).
- `completerSuggestionsList`: `ScrollView` + `LazyVStack`, each row has magnifyingglass icon + title (medium 15pt) + subtitle (secondary 13pt) + chevron.right. Tap calls `Task { await vm.selectCompleterResult(completion) }` + `isSearchFocused = false`.
- Category chips visibility gated on `vm.completerResults.isEmpty && vm.searchQuery.isEmpty` (hidden while typing).
- Existing `searchResultsList` and `aiSearchResultsList` preserved unchanged.
- AI mode toggle now also clears `vm.completerResults`.

**TripMapView.swift:**
- Pill tap: `vm.sheetDetent = .full` (was `.half`).
- `onChange(of: vm.searchQuery)`: 400ms debounce + `performMapSearch` removed, replaced with `vm.updateCompleterQuery(newValue)`. AI mode guard preserved.
