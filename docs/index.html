01 / Проблема

Планирование поездки —
это хаос

6+
Приложений на поездку
Бронирования в почте, маршрут в Google Maps, заметки в Notes, рейсы в Flightradar, погода в отдельном приложении, расходы в Excel
5ч
Среднее время планирования
Путешественник тратит 5+ часов на организацию одной поездки — поиск мест, составление маршрута, сбор чемодана, конвертация валюты
0
AI-помощников на русском
Нет ни одного travel planner на русском языке с полноценным AI, который знает твои предпочтения и адаптирует рекомендации
Ключевые боли
Разрозненная информация

Рейсы, отели, места, расходы, документы — всё в разных приложениях. Нет единого хаба для поездки.

Нет персонализации

Существующие приложения дают одинаковые рекомендации всем. Не учитывают стиль путешествий, диету, бюджет, темп.

Офлайн = слепая зона

За границей часто нет интернета. Маршруты, карты, информация о местах — недоступны без сети.

Ручной ввод рейсов

Данные о перелётах нужно вбивать вручную. Нет автоматического трекинга статуса, задержек, гейтов.

02 / Решение

в одном приложении

Почему Supabase, а не Firebase / AWS
  • $0 до 50K MAU — free tier покрывает весь первый год
  • Auth из коробки — Apple, Google, Email, OAuth (без отдельного сервиса)
  • RLS (Row-Level Security) — безопасность на уровне БД, не нужен бэкенд-сервер
  • PostgreSQL — индустриальный стандарт, легко мигрировать
  • Storage — S3-совместимый для фото (1 GB free)
  • Edge Functions — серверлесс прокси для AI API (обход блокировок из РФ)
  • $0 серверных затрат — клиент → БД напрямую через RLS
  • Realtime — для совместного планирования (v2)
vs Firebase: $0.18/GB хранения + $0.06/100K чтений — при 10K юзеров дороже на 40%.
vs AWS RDS: $15–50/мес минимум + DevOps расходы на управление.
